◈ 백트랙킹 기법
  • 문제 해결의 방법
      (1) 상태 트리를 구성
      (2) 상태 트리를 루트부터 순회하면서 최종 상태를 탐색


  • 이러한 방법의 복잡도
    • 복잡도는 문제 공간의 크기에 달려져 있다. 하지만 문제 공간의 크기는 무지무지하게 큰 경우가 보통이다. ( 정렬 문제 , 배낭 문제 )
    • 따라서 트리의 노드의 갯수가 무지하게 많고, 이를 탐색하는 것 역시 많은 시간이 걸린다.
    • 따라서 보통의 경우에 이러한 방법으로 문제를 해결하고자 시도하지는 않는다.
    • 하지만 문제의 특성상 이러한 방법 외에 다른 방법이 존재하지 않는 문제도 존재한다.
    • 이러한 종류의 문제를 위해서는 트리를 효과적으로 탐색할 수 있는 방법이 마련되어야 한다.


  • 백트랙킹 기법
      － 문제의 특성상 모든 가능성을 조사하지 않고는 해를 얻을 수가 없는 경우
      － 모든 가능성을 조직적이고도 효율적으로 조사할 수 있는 기법
      － 트리를 탐색할 때, 실현 불가능한 노드는 방문하지 않는다.
      － 예를 들어, 배낭 문제에서 배낭의 크기가 10이고 물건1의 무게가 15라고 했을 때, 물건 1을 배낭에 넣는 것도 불가능이지만, 물건 1을 넣고 또 물건 2를 더 넣는 것 역시 불가능이다. 따라서 상태 트리 순회시에 물건 1을 넣은 노드의 아래쪽의 모든 노드는 방문할 필요가 없다.
      － 트리 순회시 실현 불가능한 노드에 도달하면 그 노드의 부모 노드로 되돌아가므로 "백트랙킹"이 일어난다.


  • 알고리즘으로 본 백트랙킹 기법
      void checknode (node v )
      {
        node u ;
        if ( promising (v ) )
          if (there is a solution at v )
          write the solution;
          else
            for (each child u of v )
            checknode(u );
      }
      － 순환 호출(recursive call)은 트리에서 아래쪽으로 따라 내려가는 것을 나타내고,
      － 순환 호출되었다가 돌아가는 것이 백트랙킹이다.
