스택

1. 스택의 구조

  스택의 데이터 삽입과 제거가 한쪽 방향에서만 가능한 순서가 있는 리스트 구조이다. 삽입과 제거가 한쪽에서만 가능하므로 먼저 삽입된 데이터가 뒤에 제거되는 FILO(First-In-Last-Out) 구조이다. 그림 1은 이러한 스택의 구조를 보여주고 있다.

 

[그림 1] 스택에 데이터 A,B,C,D 삽입 및 제거 과정

 

2. 스택에 가할 수 있는 연산

 푸쉬(push) : 하나의 데이터를 스택에 추가한다.

 팝  (pop)  : 하나의 데이터를 스택에서 꺼낸다.

   push는 스택 자료구조에 하나의 데이터를 추가하는 작업이며, 보통 push된 데이터는 스택의 최상단(top)에 저장된다고 이야기한다. pop은 스택에서 하나의 데이터를 꺼내는 작업으로 스택의 최상단에 있는 데이터를 꺼내게 된다.

 

3. 스택의 특성

  가장 나중에 들어온 데이터가 먼저 나간다. (LIFO : Last In First Out)   스택은 가장 나중에 들어온 데이터가 먼저 나간다는 특성을 가진 자료구조로 컴퓨터 분야에서 문제풀이에 매우 자주 사용되는 자료구조이다. 일반적으로 함수의 호출과 복귀와, 백트래킹과 같이 자신이 거쳐왔던 길을 다시 되돌아 갈 필요가 있는 문제해결에 매우 적합한 자료구조이다. 이 스택과 함께 나중에 보게 될 큐라는 자료 구조 역시 스택처럼 데이터를 차례로 저장 했다 하나씩 꺼낸다는 점은 동일하지만, 큐는 가장 먼저 들어온 데이터가 제일 먼저 나가게 된다는 점이 다르다.

  

4. 스택의 구현

  스택을 구현하는 가장 쉬운 방법은 배열을 사용하는 것이다. 그러나 이 경우는 스택의 크기가 미리 정의되기 때문에 "overflow"가 발생할수 있다. 이 경우를 원치 않는 경우는 리스트 구조를 이용하여 스택을 작성 할 수도 있다. 이 경우는 스택의 크기가 정해지지 않기 때문에 "overflow"가 발생하지 않는다.

 (1) push : 스택에 새 원소를 추가(push)하는 것은 스택 위치(sp)에 원소를 추가하고 sp를 증가시키면 된다.

 

1) C/C++

int push(int stack[],int* sp,int ch) {

   // MAX_STACK_SIZE 는 스택이 저장할 수있는 데이터의 최대 크기를 의미한다.

   if (*sp >= MAX_STACK_SIZE)  return STACK_FULL;

   stack[sp]=ch; (*sp)++;

}

2) Pascal

Function push(stack : array ; sp : point ; ch: integer) : integer;

begin

       {MAX_STACK_SIZE 는 스택이 저장할 수있는 데이터의 최대 크기를 의미한다.}

        if sp >= MAX_STACK_SIZE then  push := STACK_FULL;

        stack[sp]:=ch;

        inc(sp);

end;

3) Basic

function push(stack(),sp,ch)

         if sp >= MAX_STACK_SIZE then

                         push=stack_full

                         exit function

          else

          stack(sp)=ch

          sp=sp+1

end function

 

(2) pop : 스택에서 원소를 pop하는 것은 스택에서 마지막에 위치한 원소를 꺼내오고 스택 위치(sp)를 감소하면 된다.

1) C/C++

int pop(int stack[],int *sp){

          if ((*sp)<=0)

                       return STACK_EMPTY;

          (*sp)--;

          return stack[sp];

}

2) Pascal

Function pop(stack : array, sp : point) : integer;

begin  

          if sp <= 0 then

                     push := STACK_EMPTY;

          sp := sp -1;

          push := stack(sp);

end;

 

3) Basic

function pop(stack(),sp)

          if sp<=0 then

                     pop=stack_empty

                     exit function

          else

           sp=sp-1

          pop=stack(sp)

end function

 

6. 스택을 이용한 응용 예제 - 연삭식 평가 (Evaluation of expression)

  연산식은 연산자의 위치에 따라 "infix", "prefix", "postfix"의 3가지로 나눌 수 있다.

   "infix" 표현 방법은 일반적으로 사용하고 있는 방식이다.

    infix notation : a + b * c - d

  "preix" 표현 방법은 연산자가 피연사자들 앞에 놓이는 경우이다.

    prefix notation : - + a * b c d

  "postfix" 표현 방법은 연산자가 피연사자들 앞에 놓이는 경우이다.

    postfix notation : a b c * + d -

  "infix"로 표현되어 있는 연산식을 "postfix"로 변경하고자 하면 어떻게 하면 될까?  이때 스택을 사용하여 구현할 수 있다. 물론 스택을 사용하지 않고도 구현할 수 있으나 그 수행 과정이 기본적으로 연산식을 두번을 읽어야 가능하기 떄문에 효율성이 떨어진다. 그러면, 어떻게 한번만 연산식을 읽어서 변환이 가능할 것인가? (이 것은 님들께 물음으로 남기겠습니다.)
