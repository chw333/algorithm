2000 Problem H - Colorful Cubes
작성일: 2001/06/17
작성자: 한상진
뛰어난 프로그래머인 빌 게임스는 부활절을 할아버지 댁에서 지내다 어릴 때 그가 가지고 놀던 나무 블럭들을 발견하게 되었다. 어릴때 그는 그 색색의 블럭으로 성과 피라밋을 짓고 놀았었다. 오늘 그가 풀려는 문제는 어릴 때 피라미드를 짓던 것보다 훨씬 복잡한 문제이다.

모든 블럭은 정육면체이며 각 면에 한 색으로 칠해져 있다. 빌은 자기가 가진 모든 블럭을 사용해서 탑을 쌓고 싶어한다. 탑은 모든 블럭을 한 줄로 쌓아올려서 만드는데, 빌은 탑을 대충 만들기는 싫어한다. 그의 원칙에 따라, 탑에 쓰이는 모든 블럭의 아래면은 그 밑에 있는 블럭의 윗면의 색하고 일치해야 한다.

입력 파일
첫줄엔 M 과 N 정수가 주어진다. M은 총 사용된 색상의 갯수이고 N은 블럭의 갯수이다.

다음 N개의 줄에 1번부터 N번까지의 블럭이 한 줄에 하나씩 보여진다. 한 블럭은 여섯 개의 색깔로 주어지며, 처음부터 앞, 뒤, 왼쪽, 오른쪽, 아래, 위 면의 색깔을 나타낸다.

출력 파일
주어진 블럭들로 빌의 원칙에 맞게 탑을 세우는 방법을 나타낸다. 모든 블럭은 한 번씩만 사용되어야 한다. 답이 여러 개 있더라도 하나만 찾으면 되고, 답이 존재한다고 가정해도 좋다.

출력 파일엔 N개의 줄이 있고 각 i번째 줄에는 아래에서 i번째로 쌓인 블럭의 정보가 주어진다. 각 정보는 일곱 개의 정수로, 첫번째에 블럭의 번호(처음에 주어진 순서대로)가 주어지고 나머지 여섯개에 앞, 뒤, 왼쪽, 오른쪽, 아래, 위의 색이 순서대로 나타난다. 블럭은 회전될 수 있음을 명심하도록 한다.

Example #1
Input file:
6 2
1 2 3 4 5 6
2 1 3 4 5 6

Output file:
1 6 5 3 4 1 2
2 6 5 3 4 2 1

Example #2
Input file:
3 3
1 2 2 2 1 2
3 3 3 3 3 3
3 2 1 1 1 1

Output file:
1 1 2 2 2 1 2
3 1 1 1 1 2 3
2 3 3 3 3 3 3
