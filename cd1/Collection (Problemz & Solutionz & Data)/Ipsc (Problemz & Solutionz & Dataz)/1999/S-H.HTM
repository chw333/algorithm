1999 Solution H - Romeo and Juliet
작성일: 2001/06/17
작성자: 한상진
(쓸데 없는 말을 너무 많이 적어 놓았기에 간단히 하겠습니다)

어떤 노드가 줄리엣이 교회가는 최단거리 위에 있고, 로미오가 경기장가는 최단거리 위에 있다면 이 노드가 둘이 만날 수 있는 가능성이 있는 곳입니다. 이 때 줄리엣과 이 노드 사이의 거리(즉, 시간이죠)와 로미오와 이 노드 사이의 거리가 같다면 이것이 답이 됩니다. 없다면 이런 노드가 존재하지 않는 것이므로 답은 -1입니다.

최단거리는 dijkstra알고리즘을 이용해 구할 수 있습니다. (물론 floyd가 구현하기 더 간편합니다만, 조금 더 비효율적입니다)


Here we provide program solving the problem as well as correct output files for both inputs.

In this problem you could use your knowledge of standard graph shortest path algorithms. Our solution works like this:

First, use Dijkstra's algorithm for computing shortest paths from JS to all other junctions. More precisely for each junction compute the shortest time it takes to get there from JS.
Second, find out, which of the nodes lie on some shortest path to JG. This can be done easily by depth-first search. We start from node JG and from each visited node v we continue to search in all its neighbours u such that if we add edge uv to the shortest path from JG to u, we get a shortest path from JG to v.
After steps 1. and 2. we know for each node whether Juliet can go through this node and if yes, at what time she passes it. Now we repeat steps 1. and 2. for Romeo (i. e. we use RS instead of JS and RG instead of JG).
Juliet and Romeo can meet if there is a node that lies on both some shortest path for Romeo and some shortest path for Juliet and both of them come to this node at the same time. Since we have computed all necessary information for all nodes, we need only to perform a simple check for each node whether it fulfils conditions set above.
Note, that the shortest path, computed here by Dijkstra's algorithms, could be computed also by Floyd-Warshall algorithm, which is easier to implement (but less efficient).
