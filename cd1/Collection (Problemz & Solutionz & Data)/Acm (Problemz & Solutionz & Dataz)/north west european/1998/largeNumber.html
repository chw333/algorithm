Large Numbers

Background

The strength of many modern cryptosystems depends heavily on the size of the keys that people use to encrypt and decrypt messages. For example, key sizes of 64 bits (ca. 1020) as used in the DES cipher can be broken within a few days with modern computer technology. At the moment, key sizes of 1024 bits (ca. 10300) are considered safe. Since usually integer arithmetics on microprocessors use integers that have at most 64 bits, everybody who wants to implement (or break) a modern cipher must implement a facility to deal with such large numbers. This is called ``multiple precision arithmetics''.

The Problem

An important operation needed in multiple precision arithmetics is multiplication. This means to calculate the product p = a * b of two numbers a and b. It is important to remember that these two numbers can be arbitrarily large.

The Input

The input will consist of a sequence of positive large number pairs. Each pair will be on a line by itself and is separated by a single blank. The numbers will consist of a (possibly very long) sequence of decimal digits without leading zeros. The end of input is marked by a pair of zeros.

In order to make input more readable, numbers may contain a backslash character meaning that they continue on the following line (see sample input). Your program must be able to handle these cases correctly.

The Output

For each input line you should output the exact product on one line by itself. The number must be broken into lines of 60 digits each if it is very long. After 60 digits you must output a backslash character followed by a newline character and continue printing digits on the next line immediately. The product should not have leading zeros. There must be one output number for each pair of input numbers other than the "0 0" that terminates the input.
