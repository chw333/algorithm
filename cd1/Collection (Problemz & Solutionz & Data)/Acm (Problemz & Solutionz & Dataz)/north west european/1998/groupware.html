Groupware

Background

In the future virtual organizations and distributed meetings will become more and more important. Groupware and CSCW (computer supported cooperative work) systems provide help for distributed teams.

CSCW systems can be distinguished in two categories depending on the kind of work mode they support: synchronous tools allow several distributed people to work together with the same documents at the same time like shared editing, while asynchronous tools help to coordinate several team members working independently with the same documents.

Besides the management of shared documents another important aspect is the communication between geographical distributed team members. In a virtual meeting every team member should get video and audio information of each other team member. Important factors for the communication are a high-quality audio and video data transfer.

The Problem

The major problem of audio and video data transfer is the broad bandwidth that is needed. Therefore it is important that the available transfer capacity of a computer network is optimally used.

For a given computer network and a given set of distributed team members the problem is to find the highest possible bandwidth such that every team member gets the video and audio data of every other member. To reduce the total amount of data a network protocol is used that allows multicast data transfer, i. e. a computer can send one data package to several recievers.

The Input

The input starts with a single line containing the number of different team settings. The input for every setting follows. The input for each team setting consists of two parts, the specification of the computer network and the set of team members.

The first line consists of two positive integers, the first specifying the number of computers / routers in the network, the second the number of connections between the network nodes. This line is followed by a triple of positive integers for every network connection. The first two give the identification numbers (the first computer within a setting has the ID 0) of the start and end network node, the third the bandwidth. Note that all connections are bi-directional and that they can transfer data in both directions simultaneously, e. g. a connection of bandwidth 10 can transfer 5 data items in both directions at the same time.

The second part of the setting starts with the number of team members in a single line. A second line contains the identification numbers of the computers the team members use.

All numbers are positive integer value which are less than 10000.

The Output

The output must contain a single integer value which represents the maximum transfer bandwidth.
