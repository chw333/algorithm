KOI 18회 중등부 1. 색종이 만들기

     

     여지껏 KOI 에 나온적 없었던 "분할정복"이 이번 KOI 에 나왔다. 이 색종이 만들기가 분할정복을 이용해 푸는 문제인데, 그리 코딩도 복잡하지 않고 간단하게 풀리는 문제이다. 우선 함수를 정의해 보자.

     Fdnc(x1, y1, x2, y2, w, b)

     위 함수는 (y1행, x1열)부터 (y2행, x2열) 까지의 사각형을 몇 개의 하얀색 색종이(w) 와 몇 개의 파란색 색종이(b)로 쪼개지는 지를 알애는 함수이다. 저기서 w 와 b 는 포인터를 써서 값을 넘겨 받도록 하자. 하얀색 색종이의 수와 파란색 색종이의 수 둘다 알아야 하는데 그 두가지 다 한꺼번에 반환을 하기 불편하기 때문이다(물론 할 수는 있다.). 자, 이제 함수를 정의 했으니 모두 끝난 것이다. 함수는 다음 총 2가지 경우로 작동한다.

       1) (y1행, x1열)-(y2행, x2열) 의 사각형이 모두 같은 색깔의 정사각형칸으로 구성되어 있는 경우
       - 모두 파란색이라면 w = 0, b = 1 를, 모두 흰색이라면 w = 1, b = 0을 반환한다. 왜냐면 (y1행, x1열)-(y2행, x2열) 이라는 사각형 자체가 하나의 색종이로 구성되기 때문이다.

       2) 그렇지 않을 경우
       - 사각형 (y1행, x1열)-(y2행, x2열)을 4개로 등분해서 각각의 사각형에 대해 Fdnc 함수를 돌려 주고 반환되는 4개의 값을 모두 더해서 현재 함수의 결과값으로 반환한다.

      함수에 인자로 w, b를 넣지 않고, int 형 함수를 써서 분할정복을 두 번 돌려 처음에는 총 색종이 수, 두번째는 하얀색 색종이 혹은 파란색 색종이 수를 반환하여 나머지 한 개의 수를 구하는 식으로 하는 방법도 있지만, 그것은 너무 느리다.

      위에 적은 방법 그대로 진행하면 된다. 이 정도 설명으로 이해가 가지 않는 다면 소스를 보면 금새 이해가 갈 것이다. 상당히 간결하게 짜여진다.
