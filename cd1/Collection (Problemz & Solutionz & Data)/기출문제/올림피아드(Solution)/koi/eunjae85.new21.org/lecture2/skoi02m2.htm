KOI 19회 중등부 2. 동전 바꿔주기

     

     음, 시간 복잡도를 어떻게 계산해야 할 지 모르겠지만, 하여간 쉽게 생각하면 효율이 그리 좋지 않은 방법이 생겨납니다. 그 방법으로 타임 오버가 나는지 않나는지는 모르겠지만, 어쨋든 그 방법을 개선하면 좀 더 좋은 방법을 얻을 수 있습니다. 여기서는 전자만을 소개하도록 하고, 후자는 스스로 생각해보시기 바랍니다. (소스는 후자의 방법으로 코딩되어 있습니다)

     price[]와 num[]을 각 화폐 단위의 가격과 그 화폐의 개수라고 정의할 때,
     부부문제 : C[i, j] = i 원을 만들 때, price[1]~price[j]를 사용하여 만들 수 있는 경우의 수

     C[i, j]에서 i 원을 만들 때, 방법은 여러 가지가 있을 수 있죠.
     price[j]를 안쓰고 i 원을 만드는 방법, price[j]를 하나 쓰고 만드는 방법, 두 개, 세 개...

     결국 C[i, j] = C[i - price[j]*0, j - 1] + C[i - price[j]*1, j - 1] + C[i - price[j]*2, j - 1] + C[i - price[j]*3, j - 1] + ...

     이런 식으로 하면 됩니다. 물론 i - price[j]*k 가 0 보다 작으면 안되겠죠.
